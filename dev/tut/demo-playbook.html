Demo Playbook

Cambridge EduX Hackathon 2026 — 5-minute presentation script & preparation guide

1. Pre-Demo Checklist

Complete every item before stepping on stage. Click to mark done.

  • Create a test class (e.g., ECON101 — Introduction to Economics)
  • Enroll 3–5 test students
  • Upload a rubric and generate an arc
  • Review and approve the arc
  • Publish the arc (scenes generated, characters assigned)
  • Run through at least 1 complete student journey (so teacher dashboard has real data)
  • Prepare a “wrong answer” script (know which misconception to trigger)
  • Test both Student PWA and Teacher Dashboard end-to-end
  • Ensure stable internet for Gemini API calls
  • Have fallback screenshots ready in case of API failure

2. 5-Minute Demo Script

1
The Problem
0:00 – 0:30

Traditional exams tell you a score, but not WHY a student struggles. Teachers grade 100 papers and still can't see the thinking process.

Assessment is disconnected from learning — students memorize, get a grade, and move on without understanding their gaps.
Misconceptions are invisible in traditional formats — a wrong answer on a multiple-choice test reveals nothing about the underlying reasoning.
Talking point: Pause briefly after the opening line. Let the problem land before transitioning: “We built Cervantes to change that.”
2
Student Experience — Live Demo
0:30 – 2:30
THE STAR
This is the WOW moment. The audience needs to feel the student experience. Walk through it live, narrating each interaction as it happens.
  1. Open Student PWA on your phone or browser — enter the Arc ID
  2. Enter Student ID — land on the first scene
  3. Show the narration fade-in and the character sprite with emotion
  4. WOW moment Give a wrong answer using a common misconception (e.g., “supply goes up when price goes down”)
  5. Show how the AI character responds with Socratic probing — it does NOT give away the answer
  6. Show character emotion change — concerned when the student is struggling, encouraging as they improve
  7. Correct the answer — show scene completion and the transition
Talking points at each step:
Step 1–2: “Students just need an Arc ID — no app download, no account creation. It is a PWA.”
Step 3: “This is a Visual Novel — narrative and character make assessment feel like a game.”
Step 4–5: “Watch what happens when the student gets it wrong — the AI doesn't correct them. It asks a follow-up question, just like a great teacher would.”
Step 6: “The character's emotions respond in real-time to the student's understanding.”
Step 7: “Every response is captured and analysed — which the teacher sees instantly.”
3
Teacher Analytics
2:30 – 4:00
  1. Switch to the Teacher Dashboard — show the class overview
  2. Show student progress cards with live status indicators
  3. Click into a student — show the reasoning trace graph (force-directed visualization)
  4. Show dimension scores with color-coded levels: mastery / developing / critical gap
Key message: “Zero grading work. AI diagnosed every misconception automatically. The teacher sees not just what students got wrong, but how they think.”
“This used to take hours of rubric grading. Cervantes does it in real-time, for every student, simultaneously.”
“The reasoning trace isn't just a score — it is a map of how the student's understanding evolved through the conversation.”
4
Architecture Flash
4:00 – 4:30

Quick high-level overview — three bullet points maximum. Do not linger here.

CurricuLLM ensures curriculum alignment — this is not random AI conversation, every question maps to learning standards
Gemini 2.5 Flash drives real-time Socratic dialogue — the AI adapts to each student's responses in the moment
Signal extraction provides 3-level diagnosis automatically from conversation transcripts
Tip: If judges seem technical, expand briefly. If not, keep this to 20 seconds and move to the close.
5
Close
4:30 – 5:00

Cervantes turns assessment into learning itself. Students learn while being assessed. Teachers get unprecedented insight — without grading a single paper.

Final beat: End confidently. Make eye contact. Do not rush to say “any questions?” — let the closing line breathe for two seconds.

4. Differentiators to Emphasize

  • 1
    Curriculum Alignment CurricuLLM ensures every question maps to learning standards — this is not generic AI chat.
  • 2
    Anti-Cheating Design Each student gets unique character variants with different names and personalities — no two experiences are identical.
  • 3
    3-Level Diagnosis mastery / revised_with_scaffolding / critical_gap — not just right or wrong.
  • 4
    Reasoning Visualization Force-directed graph shows concept mastery relationships — teachers see how students think, not just what they scored.
  • 5
    PWA Mobile-First Students access the assessment on any device — no app install, no download, works offline after first load.

5. Do’s and Don’ts

Do Don’t
Pre-load a published Arc before the demo Wait for AI generation live (too slow, risky)
Use the Economics course for the demo Use technical jargon with non-technical judges
Show the student experience FIRST Start with architecture slides
Emphasize “dialogue-based assessment” Over-explain Visual Novel mechanics
Show the mobile PWA experience Show placeholder pages (log / calendar / map)
Prepare a wrong-answer script in advance Wing the student responses during the demo
Have backup screenshots ready on standby Rely solely on live demo with no safety net

6. Backup Plan

If something fails during the live demo, switch to the fallback immediately. Do not debug on stage.

Gemini API slow Switch to pre-recorded screenshots or screen-capture video of a successful student journey.
Login fails Use E2E test mode — set NEXT_PUBLIC_E2E_TEST=true to bypass authentication entirely.
Student dashboard down Skip to Teacher Dashboard with pre-loaded data — the analytics still tell the full story.
WiFi unstable Use phone hotspot as backup. The Student PWA works offline after initial load (service worker cache).
Golden rule: If the live demo breaks, acknowledge it casually (“Let me show you what this looks like”), switch to screenshots, and keep talking. Never apologize twice or try to fix it live.
