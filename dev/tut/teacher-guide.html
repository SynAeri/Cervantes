The Teacher Journey

A step-by-step walkthrough of the professor experience in Cervantes — from authentication through real-time student monitoring and deep learning analytics.

lock_open

Login

/login

The teacher journey begins at the Cervantes login portal. Professors authenticate with email/password, Google OAuth, or Microsoft OAuth — all powered by Firebase Auth.

Upon first sign-in the frontend automatically calls /api/auth/register to register the user with role: professor in the backend.

  • Branded "Cervantes" logo with terracotta accent
  • Email + password form with visibility toggle
  • Side-by-side Google and Microsoft OAuth buttons
  • Toggle between Sign In and Sign Up flows
  • Animated WheatField SVG decoration behind the card
info
The WheatField animation renders subtle, wind-swayed wheat stalks across the bottom of the viewport — a visual nod to the La Mancha landscape that underpins the Cervantes design system.

Create Assessment Arc

/class/[classId]/arc/new

The arc creation page guides the professor through a 3-step workflow:

1
Upload
2
Generating
3
Approved

Upload — A drag-and-drop zone accepts assessment rubrics in PDF, DOCX, or TXT format. The professor can also click "Browse Files" to use a file picker.

Generating — Once uploaded, the system enters an animated generation phase. A spinner rotates while status messages pulse through the stages:

  • description Parsing rubric
  • target Extracting objectives
  • psychology Identifying misconceptions
  • auto_stories Generating narrative
  • person Creating characters
  • movie Planning scenes
  • check_circle Finalizing arc
bolt
Behind the scenes: CurricuLLM parses the rubric document, extracts learning outcomes, key concepts, and common misconceptions. Gemini then plans the arc structure — determining scene order, types, and character assignments. Generation typically takes 30–60 seconds.

ArcReviewModal

When generation completes, the ArcReviewModal slides up over the page. It presents the full arc structure for the professor to inspect before committing.

The modal displays a list of scenes, each showing:

  • Character assigned to the scene

  • Misconception target — the common error probed
  • Editable learning objectives for fine-tuning

publish

Publish to Students

/class/[classId]

After approval, the Arc Status Box shows a "Publish Scenes" button. Clicking it triggers the full content generation pipeline:

  • System generates complete VN scene content for every scene in the arc
  • Character pools are created — 2–3 name variants per scene to add variety
  • Random character variants are assigned to each enrolled student
  • Arc status transitions to "published"
  • A student access link becomes available in the Arc Status Box
link
The student link format is student-portal/{arc_id} — the professor can share this with the class. Each student sees uniquely assigned character variants while experiencing the same underlying assessment content.

Real-time Monitoring

/class/[classId]

Once published, the Student Performance table becomes a live dashboard. It polls the backend every 10 seconds via useClassArcProgress, merging fresh progress data with the student list.

Each student row displays:

  • Name, progress bar, and percentage
  • Color-coded dimension scores for each learning objective
  • Arc status indicator
