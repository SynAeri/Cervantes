3. Getting Started

1

Clone & Environment Setup

Clone the repository and create your environment file with the required secrets.

bash Clone
git clone https://github.com/SynAeri/Cervantes.git cd Cervantes

Create .env in project root (or app/backend/.env):

env .env
DATABASE_URL=postgresql://postgres:postgres@db:5432/cervantes GEMINI_API_KEY=your_gemini_api_key CURRICULLM_API_KEY=your_curricullm_api_key JWT_SECRET=your_jwt_secret JWT_ALGORITHM=HS256 JWT_EXPIRATION_HOURS=168 GOOGLE_APPLICATION_CREDENTIALS=app/backend/cervantes-service-account.json
2

Start Backend

Docker Compose runs PostgreSQL 16 (port 5432) and the FastAPI server (port 8000).

bash Backend
docker compose up --build # Backend at http://localhost:8000 # Health check: curl http://localhost:8000/health
3

Start Student Dashboard

The student-facing PWA runs on port 3000.

bash Student Dashboard
cd student-dashboard npm install npm run dev # → http://localhost:3000
Note Uses webpack (not Turbopack) for stability with Next.js 16.
4

Start Teacher Dashboard

The teacher analytics dashboard runs on port 3001.

bash Teacher Dashboard
cd teacher-dashboard npm install npm run dev # → http://localhost:3001

4. Environment Variables Reference

Variable Required Where Description
GEMINI_API_KEY Yes Backend Google Gemini 2.5 Flash API key
CURRICULLM_API_KEY Yes Backend CurricuLLM curriculum engine key
DATABASE_URL Yes Backend PostgreSQL connection string
JWT_SECRET Yes Backend JWT signing secret
GOOGLE_APPLICATION_CREDENTIALS Yes Backend Path to Firebase service account JSON
NEXT_PUBLIC_API_URL No Frontends Backend URL (default: http://localhost:8080)
NEXT_PUBLIC_E2E_TEST No Frontends Enable E2E test mode (bypasses Firebase auth)

6. Deployment

Backend → GCP Cloud Run

Region: australia-southeast1 • Project: cervantes-caebc • Service: cervantes-backend
bash Deploy
./scripts/deploy.sh --env-file app/backend/.env

Frontends → Firebase App Hosting

Auto-deploy on push to main
Configured in student-dashboard/apphosting.yaml and teacher-dashboard/apphosting.yaml. No manual deploy step required — pushes to the main branch trigger automatic builds and deployment.

7. Common Issues

Pydantic Settings rejects extra env vars
Remove extra variables from .env or add model_config = SettingsConfigDict(extra="ignore") to your settings class.
Missing PyPDF2 / python-docx
These are optional processing dependencies. Install with pip install PyPDF2 python-docx.
GOOGLE_APPLICATION_CREDENTIALS path not found
The path must be absolute or relative to the working directory of the running process. Inside Docker, the working directory is /app.
Port conflicts between local and Cloud Run defaults
Backend uses port 8000 (Docker) or 8080 (Cloud Run). Frontend defaults expect 8080 — set NEXT_PUBLIC_API_URL=http://localhost:8000 for local dev.
